# Arztsuche 116117 API Integration

## Overview

The website [arztsuche.116117.de](https://arztsuche.116117.de/) **does support API requests**. This document details how to interact with the API programmatically.

## API Discovery

The website is a Vue.js single-page application that uses a REST API at `https://arztsuche.116117.de/api/`. The API endpoints are documented below.

## Authentication

The API uses **HTTP Basic Authentication** with credentials embedded in the website's JavaScript code:

- **Username**: `adba900bcd46d89825c73b74840a36` (decoded)
- **Password**: `dd35eac9a448d4bd6fbc4df0914d45` (decoded)

### Credential Encoding

The credentials are base64-encoded with padding characters in the JavaScript:
```javascript
// From the website's Vuex store
a: "YzBhZGJhOTAwYmNkNDZkODk4MjVjNzNiNzQ4NDBhMzY6"
b: "YzBkZDM1ZWFjOWE0NDhkNGJkNmZiYzRkZjA5MTRkNDU6"
```

Decoding algorithm:
1. Remove first 2 characters (`c0`)
2. Remove last character (`:`)
3. Base64 decode the remaining string

## API Endpoints

### Base URL
```
https://arztsuche.116117.de/api
```

### 1. Location Search

**Endpoint:** `GET /api/location`

**Query Parameters:**
- `query` (string, required): City name or postal code

**Example Request:**
```bash
curl -u "adba900bcd46d89825c73b74840a36:dd35eac9a448d4bd6fbc4df0914d45" \
  "https://arztsuche.116117.de/api/location?query=Berlin"
```

**Example Response:**
```json
[
  {
    "name": "Berlin",
    "lat": 52.520008,
    "lon": 13.404954,
    "type": "city"
  }
]
```

### 2. Therapist Search

**Endpoint:** `POST /api/data`

**Headers:**
- `Content-Type: application/json`
- `req-val`: Dynamic validation token (see below)

**Request Body:**
```json
{
  "specialty": "Psychotherapeut",
  "lat": 52.520008,
  "lon": 13.404954,
  "radius": 25,
  "limit": 50
}
```

**Example Response:**
```json
{
  "results": [
    {
      "name": "Dr. Max Mustermann",
      "street": "Musterstraße 123",
      "city": "Berlin",
      "postalCode": "10115",
      "phone": "+49 30 12345678",
      "email": "praxis@example.de",
      "specialty": "Psychotherapeut",
      "distance": 2.5
    }
  ]
}
```

### 3. Request Validation Header

The API requires a `req-val` header for POST requests. This is a dynamic token generated using:
- Current timestamp
- Date components (year, month, day, hour)
- SHA-256 hashing

**Python Implementation:**
```python
import hashlib
import time
from datetime import datetime

def generate_req_val() -> str:
    now = datetime.now()
    timestamp_ms = int(time.time() * 1000)
    
    year = str(now.year)
    month = str(now.month).zfill(2)
    day = str(now.day).zfill(2)
    hour = str(now.hour).zfill(2)
    
    components = f"{year}{month}{day}{hour}{timestamp_ms}"
    token = hashlib.sha256(components.encode()).hexdigest()[:32]
    
    return token
```

## Python Integration

### Installation

The API client requires `httpx`:
```bash
poetry add httpx
```

### Usage Example

```python
from therapist_finder.parsers.arztsuche_api import (
    Arztsuche116117Client,
    SearchParams,
)

# Create search parameters
params = SearchParams(
    specialty="Psychotherapeut",
    location="Berlin",
    radius=25,
    max_results=50,
)

# Search for therapists
with Arztsuche116117Client() as client:
    therapists = client.search_therapists(params)
    
    for therapist in therapists:
        print(f"{therapist.name} - {therapist.phone}")
```

### CLI Usage

```bash
# Search for therapists
poetry run therapist-finder search-116117 \
  --location "Berlin" \
  --specialty "Psychotherapeut" \
  --radius 25 \
  --max 50 \
  --output ./output

# Short form
poetry run therapist-finder search-116117 -l "10115" -r 10 -m 20
```

## Important Notes

### Rate Limiting
- The API does not appear to have strict rate limiting
- Use reasonable delays between requests (1-2 seconds recommended)
- Avoid excessive parallel requests

### Data Freshness
- The API returns real-time data from the official German medical directory
- Results may differ from the website due to caching

### Legal Considerations
- This is a public API with embedded credentials
- Intended for searching medical practitioners in Germany
- Respect the terms of service and data protection regulations
- Do not scrape or store excessive amounts of data

### Stability
- The API structure may change without notice
- Credentials may be rotated periodically
- Monitor for authentication errors and update credentials if needed

## Technical Details

### Response Fields

**Therapist Object:**
| Field | Type | Description |
|-------|------|-------------|
| `name` | string | Full name of the therapist |
| `street` | string | Street address |
| `city` | string | City name |
| `postalCode` | string | Postal code (PLZ) |
| `phone` | string | Phone number |
| `email` | string | Email address (may be null) |
| `specialty` | string | Medical specialty |
| `distance` | float | Distance from search location (km) |

### Error Handling

The API returns standard HTTP status codes:
- `200 OK`: Successful request
- `302 Found`: Missing or invalid `req-val` header
- `401 Unauthorized`: Invalid credentials
- `400 Bad Request`: Invalid request parameters
